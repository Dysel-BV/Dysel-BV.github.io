# Dysel BC 28.0.202605

| Case No. | Module | Comment |
| --- | --- | --- |
| 64002 | General | The New Extended Text feature is now fully implemented and has replaced the old extended text. |
| 66483 | General | Fixed an issue on the Sales Order Card page where the ELC Doc. Type field displayed blank for orders created from a Consolidated Order or Work Order. The field now correctly shows all assigned values. The field is read-only; the value can be changed via the page actions Set to Empty, Set to Parts, and Set to Equipment. Changing the value via these page actions is only allowed if the Document No. W1 ELC field is empty. |
| 68213 | General | Contact lookup and auto-populate now work correctly on Maintenance Contracts. Selecting a sell-to or bill-to contact automatically fills the contact name, phone number, and e-mail. Lookups are filtered by the relevant customer. All 8 contact fields are personalizable (hidden by default). |
| 68345 | General | A new Contact No. field has been added to the ship-to address card. Selecting a contact automatically fills in the contact name, phone number, and e-mail from the contact card. Clearing the field removes those values. Phone No. and E-Mail are now always visible on the card. |
| 68470 | General | Implemented performance enhancements and locking mitigation strategies in certain heavily used processes. |
| 68574 | General | Features **Rental Credit Wizard** and **Service Action Dispatch Board Work Order Creation Wizard** have been activated by default in Dysel Feature Management and are now part of the standard methodology. |
| 68608 | General | Introduced changes on the maintenance contract to help reduce the number of SQL calls when opening a maintenance contract card in an effort to increase performance. |
| 68691 | General | Fixed issues with upgrade reports (11021796 and 11021598) that were causing errors in the Job Queue. |
| 68535 | Equipment | Fixed an issue with undoing posted purchase receipts on equipment object lines. Previously, undo could leave an incorrect residual purchase value. This is now corrected. |
| 68730 | Equipment | Service Actions now use the correct date when posting a Work Order. |
| 67814 | Parts | We improved item creation from the Catalogue Card when item templates are involved. The flow now: 1. Resolves template selection before creating the item record. 2. Applies the template after item creation in a safe order. 3. Applies the catalogue Unit of Measure through standard field validation to avoid duplicate Item Unit of Measure records. |
| 68402 | Parts | We introduced a new utility report: Clean up Requisition Worksheet. This report will also be added to the Job Queue, where you only need to complete the setup and enable the job queue if you want to clean up your requisition worksheet periodically. |
| 68759 | Parts | When using the Create Item from Catalogue Entry action, an error occurred during the processing of the Item Category Code. The system attempted to save the item record at an incorrect point in the creation flow, causing the action to fail entirely. This issue has been resolved. The item creation process now correctly assigns the Item Category Code and its associated Price Group without triggering a premature save. |
| 65214 | Service | Maintenance Quotes can now be converted to Lost Deals. A lost deal is created per line; if the quote has one line the quote is deleted, otherwise only the processed line is removed. Existing Rental and Sales lost deal flows are unchanged. |
| 68261 | Service | Improved work order line numbering logic for job code and job code setup BOM lines inserted from the service type. All line numbers now follow Business Central standards, ensuring correct sequencing and preventing duplication in work order processing. |
| 68571 | Service | Fixed the display of the work order number on the Service Actions FastTab in the Work Order Card. |
| 68625 | Service | When recreating a work order, for example in the Credit This Invoice process, the extended text for the work order reporting codes was already copied to the new lines. We have now ensured that the old extended text for the closed work order, stored in the extended text archive, is no longer deleted. |
| 68640 | Service | When creating work orders via the Service Action Dispatch Board (SADB) Wizard, users are now prompted once after creation to assign preferred resources as allocated resources for all applicable work orders. The generated Excel report now also includes a **Location Name** column alongside the existing location type and number. |
| 68755 | Service | Fixed an issue that caused incorrect behavior with job code lines that were automatically inserted via the service type. |
| 67959 | Rental | Introduced recurring rental charge invoicing. Charges can now remain standard, be postponed temporarily, or be set to recur during future invoice runs. Users should review the new Charge Invoicing field on charges and rental contract charge lines. If Postpone is used, a Postponement Reason still needs to be maintained. |
| 68508 | Rental | Fixed object line posting groups on recreated source orders after credit posting. Processed related document numbers are now persisted from the return document creation flow, passed through pre-posting, and used to update object sales lines after Sales-Post when the equipment posting status has been reverted from Sold. |
| 68753 | Rental | Fixed a bug where crediting a rental sales invoice and recreating the source document resulted in the wrong Gen. Prod. Posting Group on the recreated sales order line. The root cause was Validate(Quantity) overwriting posting groups derived from the Equipment Prod. Post. Setup. The fix restores the correct rental posting groups after validation. |
| 68758 | Rental | Resolved bug that prevented lost deal creation for model lines. |
| 68587 | Purchase | We fixed an issue where Source Code could be set incorrectly to Sales in purchase-invoice related posting scenarios. |
| 68449 | Mobile Field Service | Work Order Post Usage now works correctly with DMB Item Status "Next Visit". |
| 68626 | Mobile Field Service | Customer View for Work Order Overview is now sorted alphabetically |
| 68627 | Mobile Field Service | The Home menu no longer flashes when the Customer page is opened from the Work Order page. |
